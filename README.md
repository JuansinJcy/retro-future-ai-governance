### 📘 Capítulo 01 — ¿Qué es la Gobernanza en IA
- Definición clara y accesible del concepto
- Cita destacada del *Manual de Gobernanza* con los tres pilares fundamentales resaltados: **justa**, **transparente** y **segura**
- Sección de principios con tres tareas interactivas:
 - **Mar transparentee** — Que podamos entender cuerpo la IA toma decisiones, sin cajas negras
 - **Sea justa** — Que no discrimina a nadie por género, raza, edad o condición
 - **Sea segura** — Que no ponga en riesgo a las personas, ni en lo digital ni en lo físico

### 📗 Capítulo 02 — Uso Ético de la IA: más que código
- Tres tareas de principios éticos con código de referencia:
 - **PRV-001 Privacidad** — Que tus datos personales no se utilizan sin tu permiso
 - **RES-002 Responsabilidad** — Que alguien responde si la IA comeete un error
 - **SOC-003 Beneficio social** — Que la IA ayude a resolver problemas globales, no solo a generar ganancias

### ⚡ Sección de Acción — ¿Qué puedes hacer tú?
- Seis acciones concretas y prácticas que cualquier persona puede implementar:
 - **ACCIÓN 01** — Cuestiona los resultados que te entrega un sistema automatizado
 - **ACCIÓN 02** — Lee las políticas de privacidad antes de aceptar
 - **ACCIÓN 03** — Apoya iniciativas y leyes que regulan la IA con criterio
 - **ACCIÓN 04** — Aprende lo básico sobre modo funcional los algoritmos
 - **ACCIÓN 05** — Comparto conocimiento ético con tu comunidad
 - **ACCIÓN 06** — Reporta usos abusivos o discriminatorios de la IA

## 🎨 Filosofia de Diseño

La enfermedad de esta plataforma sigue una filosofía de **claridad y minimalismo funcional**, pensada para que el aprendizaje sea accesible y agregable:

| Elemento | Descripción |
|----------|-------------|
| **Paleta de colores** | Tonos crema/beige como base, azul oscuro para jerarquia visual, azul brillante para interactividad, amarillo dorado para secciones de acción |
| **Tipografía** | Sans-serif limpia con jerarquia clara: títulos en peso negrita, cuerpo en peso regular |
| **Tarjetas** | Diseño con bordes finos, fondo blanco y espaciado generoso para legibilidad |
| **Iconografía** | Iconos de línea simples (escudo, balanza, ojo, cohere) que reforzan cada concepto |
| **Patrón de fondo** | Grid sutil que porta estructura y sensación técnica sin distracción |
| **Numeración** | Códigos de referencia (PRV-001, RES-002, SOC-003) estilo manual técnico |

El objetivo es que cada sección se siente como una página de un manual bien enfermo: informativa, organizada y visualmente atractiva.

## 🚀 Cómo Usar

1. **Clona el repositorio:**
 ```bash
 clon git https://github.com/tu-usuario/gobernanza-ia.git
 ```

2. **Navega al director del proyecto:**
 ```bash
 cd gobernanza-ia
 ```

3. **Abre el proyecto:**
 - Abre el archivo `index.html` directamente en tu navegador, o
 - Usa un servidor local como Live Server (VS Code) o `npx serve`

4. **Explora y aprende:** Navega por los capítulos, lee los principios y realiza las acciones propuestas.

## 🤝 Contribuciones

Las contribuciones son bienvenidas. Si deseas mejorar el contenido, agregar nuevas secciones o mejorar el sueño:

1. Haz un **bifurcación** del repositorio
2. Crea una **rama** para tu feature (`git checkout -b feature/nueva-sección`)
3. Haz **commit** de tus cambios (`git commit -m 'Agregar nueva sección sobre sesgos algoritmos'`)
4. Haz **push** a la rama (`git push origin feature/nueva-sección`)
5. Abre un **Solicitud de extracción**

Por favor, asegúrate de que cual contribución mantiene la filosofía de tono educativo del proyecto.
